Documentation

Everything you need to know about using Clonix to archive HTML websites.

Supported

  • HTML websites & pages
  • CSS stylesheets
  • JavaScript files
  • Images (PNG, JPG, SVG, WebP, GIF, ICO)
  • Web Fonts (WOFF, WOFF2, TTF, EOT, OTF)
  • PDF documents

Not Supported

  • React / Vue / Svelte apps
  • Next.js / Nuxt applications
  • Angular single-page apps
  • Backend / server-side code
  • Databases & dynamic content

Frequently Asked Questions

Clonix starts by fetching the homepage of the provided URL. It then parses the HTML to discover internal links, stylesheets, scripts, and images. It recursively visits internal pages, downloads all discovered assets, rewrites URLs to local relative paths, and packages everything into a downloadable ZIP archive.
Clonix works best with traditional HTML-based websites. Static sites, documentation pages, blogs, and portfolios are ideal candidates. Websites that rely heavily on JavaScript frameworks like React, Angular, or Vue for rendering content may not be fully captured since their content is generated dynamically.
By default, Clonix will crawl up to 100 pages per website. This limit helps prevent extremely long processing times and ensures a reasonable download size. For most websites, this covers all primary content.
For static HTML websites, the cloned version should look very similar to the original. All CSS, images, and JavaScript files are downloaded and URLs are rewritten to work locally. However, features that depend on a server (like forms, search, or APIs) will not function in the offline copy.
ZIP files are stored on the server for 30 minutes after generation. Make sure to download your archive promptly. After 30 minutes, the file is automatically deleted to save server resources.
Clonix is designed for archiving websites you own or have permission to copy. Always respect copyright, terms of service, and robots.txt directives. Using this tool to clone websites without authorization may violate applicable laws and regulations.
